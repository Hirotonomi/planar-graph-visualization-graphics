\section{Cel projektu}

Celem projektu jest stworzenie aplikacji w~języku Java z~graficznym interfejsem użytkownika opartym na bibliotece Swing, umożliwiającej wczytywanie, wizualizację oraz interaktywną edycję układu wierzchołków grafów planarnych. Aplikacja wywołuje zewnętrzny binarny silnik obliczeniowy w~języku C (opracowany przez zespół Poliny Vyshni i~Andreia Udavichenki) w~celu wyznaczenia współrzędnych wierzchołków przy użyciu wybranego algorytmu, a~następnie prezentuje wyniki graficznie. Możliwe jest również pominięcie etapu obliczeń i~bezpośrednie wczytanie wcześniej przygotowanego pliku pozycji. Interfejs użytkownika jest w~całości w~języku polskim.

% ---------------------------------------------------------------
\section{Opis funkcjonalności}

Aplikacja udostępnia następujące funkcje:

\begin{itemize}
    \item wczytywanie opisu grafu (lista krawędzi) z~pliku tekstowego za pomocą okna dialogowego,
    \item wywołanie zewnętrznego binarnego silnika obliczeniowego jako procesu potomnego z~odpowiednimi argumentami (ścieżka wejściowa, ścieżka wyjściowa, wybrany algorytm, format zapisu),
    \item wczytanie gotowego pliku pozycji bez uruchamiania obliczeń -- w~formacie tekstowym lub binarnym,
    \item automatyczne nazwanie pliku wyjściowego na podstawie nazwy pliku wejściowego (przyrostek \texttt{\_positions}),
    \item wyświetlenie grafu w~panelu wizualizacji po zakończeniu obliczeń lub wczytaniu pliku pozycji,
    \item interaktywna edycja pozycji wierzchołka przez przesuwanie go przyciskami kierunkowymi z~konfigurowalnym krokiem przesunięcia,
    \item interaktywna edycja pozycji wierzchołka metodą przeciągnij i~upuść (\textit{ang.~drag \& drop}) bezpośrednio w~panelu wizualizacji,
    \item zmiana skali i~pozycji widoku przy użyciu suwaka powiększenia oraz przycisków pomocniczych,
    \item przełączanie widoczności etykiet wierzchołków i~wag krawędzi,
    \item czytelne komunikaty o~błędach wyświetlane w~oknach dialogowych interfejsu graficznego.
\end{itemize}

% ---------------------------------------------------------------
\section{Interfejs graficzny}

Okno główne aplikacji podzielone jest na cztery obszary: poziomy pasek menu na górze okna, lewy panel boczny zajmujący około 25--30\% szerokości okna, prawy obszar wizualizacji grafu oraz wąski pasek stanu na dole okna. Wszystkie napisy, etykiety, przyciski i~komunikaty są wyłącznie w~języku polskim.

\subsection{Pasek menu}

Pasek menu jest stałym, poziomym elementem umieszczonym na górze okna. Zawiera trzy opcje, których kliknięcie przełącza zawartość lewego panelu bocznego:

\begin{itemize}
    \item \textbf{Ustawienia grafu} -- wyświetla zakładkę konfiguracji wczytywania i~obliczeń,
    \item \textbf{Edycja} -- wyświetla zakładkę ręcznej edycji pozycji wierzchołków,
    \item \textbf{Ustawienia widoku} -- wyświetla zakładkę ustawień wyświetlania i~powiększenia.
\end{itemize}

Pasek menu nie zawiera rozwijanych podmenu -- każda opcja działa jako przycisk przełączający stan panelu bocznego. W~danym momencie aktywna jest dokładnie jedna zakładka.

% ---------------------------------------------------------------
\subsection{Panel boczny (lewy)}

Panel boczny zajmuje około 25--30\% szerokości okna. Jego zawartość zmienia się w~zależności od wybranej opcji w~pasku menu.

\subsubsection{Zakładka: Ustawienia grafu}
\label{sec:panel-graph}

Zakładka służy do skonfigurowania danych wejściowych, wyboru sposobu wyznaczania pozycji i~uruchomienia procesu. Elementy ułożone od góry do dołu:

\begin{enumerate}

    \item \textbf{Plik wejściowy} -- pole tekstowe ze ścieżką do pliku oraz przycisk \textit{Przeglądaj} otwierający systemowe okno dialogowe wyboru pliku. Akceptowany format: plik tekstowy z~listą krawędzi (sekcja~\ref{sec:format-input}). Wartość domyślna: \texttt{input.txt}.

    \item \textbf{Wybór trybu wyznaczania pozycji} -- grupa przycisków opcji (\textit{radio button}) podzielona na dwie sekcje wizualnie oddzielone separatorem:

    \textit{Sekcja 1 -- Obliczanie (wywołuje silnik C):}
    \begin{itemize}
        \item \textbf{Fruchterman-Reingold} -- algorytm siłowy oparty na symulacji fizyki; węzły odpychają się jak cząstki naładowane, a~krawędzie przyciągają je jak sprężyny; iteracyjne chłodzenie zapewnia równomierny i~czytelny układ (opcja domyślna),
        \item \textbf{Triangulacja} -- algorytm geometryczny gwarantujący brak przecięć krawędzi; wierzchołki są rozmieszczane z~użyciem reguł triangulacji.
    \end{itemize}
    Po wybraniu jednej z~tych opcji aktywne stają się pola \textit{Katalog wyjściowy} i~\textit{Format zapisu}, a~przycisk \textit{Wykonaj} uruchamia silnik C.

    \textit{Sekcja 2 -- Wczytaj gotowy plik pozycji (pomija obliczenia):}
    \begin{itemize}
        \item \textbf{Plik tekstowy (.txt)} -- wczytuje plik w~formacie tekstowym (sekcja~\ref{sec:fmt-txt}); obok przycisku opcji wyświetlane jest pole ścieżki z~przyciskiem \textit{Przeglądaj},
        \item \textbf{Plik binarny (.bin)} -- wczytuje plik w~formacie binarnym (sekcja~\ref{sec:fmt-bin}); obok przycisku opcji wyświetlane jest pole ścieżki z~przyciskiem \textit{Przeglądaj},
    \end{itemize}
    Po wybraniu jednej z~opcji wczytywania pola \textit{Katalog wyjściowy} i~\textit{Format zapisu} są wyszarzone (nieaktywne). Kliknięcie przycisku \textit{Wykonaj} wczytuje wskazany plik i~wyświetla graf bez wywoływania silnika C.

    \item \textbf{Katalog wyjściowy} -- pole tekstowe ze ścieżką do katalogu zapisu oraz przycisk \textit{Przeglądaj}. Aktywne tylko w~trybie obliczania. Wartość domyślna: podkatalog \texttt{positions} w~katalogu programu.

    \item \textbf{Format zapisu} -- dwa niezależne pola wyboru, aktywne tylko w~trybie obliczania:
    \begin{itemize}
        \item \textit{Binarny (.bin)} -- włącza zapis w~formacie binarnym,
        \item \textit{Tekstowy (.txt)} -- włącza zapis w~formacie tekstowym.
    \end{itemize}
    Możliwe jest zaznaczenie obu formatów jednocześnie. Jeśli nie zaznaczono żadnego, program domyślnie zapisuje plik binarny. Nazwa pliku wyjściowego tworzona jest automatycznie jako nazwa pliku wejściowego z~przyrostkiem \texttt{\_positions} (np.~\texttt{graf\_positions.bin}).

    \item \textbf{Przycisk \textit{Wykonaj}} -- działanie zależy od wybranego trybu:
    \begin{itemize}
        \item tryb obliczania: aplikacja wywołuje silnik C z~odpowiednimi argumentami; po zakończeniu wyświetla graf i~aktualizuje pasek stanu,
        \item tryb wczytywania: aplikacja odczytuje wskazany plik i~wyświetla graf bez wywoływania silnika C.
    \end{itemize}
    Jeśli wymagane pole ścieżki jest puste, wyświetlany jest komunikat o~błędzie i~żadna akcja nie jest podejmowana.

\end{enumerate}

% - - - - - - - - - - - - - - - - - - - - - -
\subsubsection{Zakładka: Edycja}
\label{sec:panel-edit}

Zakładka umożliwia korektę pozycji wierzchołków bez ponownego uruchamiania silnika obliczeniowego. Obsługiwane są dwa niezależne mechanizmy edycji.

\vspace{0.5em}
\textbf{Edycja przyciskami kierunkowymi:}

\begin{enumerate}
    \item \textbf{Wybór wierzchołka} -- lista rozwijana (\textit{combo box}) z~identyfikatorami wszystkich wierzchołków aktualnie wczytanego grafu. Aktywna dopiero po wczytaniu lub obliczeniu pozycji.
    \item \textbf{Krok przesunięcia} -- pole numeryczne określające, o~ile jednostek przesuwa się wierzchołek przy jednym kliknięciu przycisku. Wartość domyślna: \texttt{1.0}.
    \item \textbf{Przyciski kierunkowe} -- cztery przyciski ułożone w~kształt krzyża:
    \begin{itemize}
        \item \textbf{$\uparrow$ Góra} -- przesuwa wybrany wierzchołek w~górę o~wartość kroku,
        \item \textbf{$\downarrow$ Dół} -- przesuwa wybrany wierzchołek w~dół o~wartość kroku,
        \item \textbf{$\leftarrow$ Lewo} -- przesuwa wybrany wierzchołek w~lewo o~wartość kroku,
        \item \textbf{$\rightarrow$ Prawo} -- przesuwa wybrany wierzchołek w~prawo o~wartość kroku.
    \end{itemize}
    Każde kliknięcie natychmiast odświeża panel wizualizacji.
    \item \textbf{Przycisk \textit{Zatwierdź do pliku}} -- zapisuje bieżący stan wszystkich współrzędnych do pliku wyjściowego. Silnik C nie jest uruchamiany ponownie.
\end{enumerate}

Przyciski kierunkowe i~\textit{Zatwierdź do pliku} są nieaktywne dopóki nie wybrano wierzchołka z~listy.

\vspace{0.5em}
\textbf{Edycja metodą przeciągnij i~upuść (\textit{drag \& drop}):}

Mechanizm dostępny bezpośrednio w~panelu wizualizacji, niezależnie od aktywnej zakładki. Realizowany przez implementację interfejsów \texttt{MouseListener} i~\texttt{MouseMotionListener} biblioteki Swing:

\begin{itemize}
    \item \texttt{mousePressed} -- kliknięcie i~przytrzymanie na wierzchołku zaznacza go i~rozpoczyna tryb przeciągania,
    \item \texttt{mouseDragged} -- wierzchołek podąża za kursorem; panel wizualizacji odświeżany jest na bieżąco,
    \item \texttt{mouseReleased} -- zwolnienie przycisku kończy przeciąganie; nowe współrzędne są zapisywane do pliku wyjściowego i~synchronizowane z~listą w~zakładce \textit{Edycja}.
\end{itemize}

Podczas przeciągania wierzchołek jest wizualnie wyróżniony. Kliknięcie w~obszar poza wierzchołkiem nie wywołuje żadnej akcji.

% - - - - - - - - - - - - - - - - - - - - - -
\subsubsection{Zakładka: Ustawienia widoku}

Zakładka steruje sposobem wyświetlania grafu. Wszystkie zmiany są natychmiastowe.

\begin{enumerate}
    \item \textbf{Pokaż identyfikatory wierzchołków} -- pole wyboru; domyślnie włączone. Gdy aktywne, przy każdym wierzchołku wyświetlana jest jego etykieta.
    \item \textbf{Pokaż wagi krawędzi} -- pole wyboru; domyślnie wyłączone. Gdy aktywne, przy każdej krawędzi wyświetlana jest jej waga.
    \item \textbf{Powiększenie} -- suwak zmieniający skalę widoku w~zakresie 50\%--300\%. Obok suwaka wyświetlana jest bieżąca wartość procentowa.
    \item \textbf{Przycisk \textit{Resetuj zoom}} -- przywraca skalę 100\%.
    \item \textbf{Przycisk \textit{Wyśrodkuj}} -- wyśrodkowuje graf w~panelu wizualizacji przy bieżącej skali.
\end{enumerate}

% ---------------------------------------------------------------
\subsection{Panel wizualizacji (prawy)}

Prawy obszar okna wypełnia niestandardowy komponent Swing (\texttt{PanelGrafu}) z~własną implementacją metody \texttt{paintComponent()}. Po uruchomieniu aplikacji obszar ma czarne tło i~jest pusty -- graf pojawia się dopiero po kliknięciu przycisku \textit{Wykonaj}.

Komponent rysuje:
\begin{itemize}
    \item krawędzie grafu jako odcinki proste łączące odpowiednie wierzchołki,
    \item wierzchołki jako wypełnione okręgi; wierzchołek zaznaczony (podczas edycji lub przeciągania) jest wyróżniony innym kolorem,
    \item opcjonalne etykiety identyfikatorów wierzchołków,
    \item opcjonalne etykiety wag krawędzi.
\end{itemize}

Współrzędne wierzchołków są automatycznie skalowane do rozmiaru komponentu z~uwzględnieniem bieżącego poziomu powiększenia. Komponent obsługuje zdarzenia myszy realizujące mechanizm przeciągnij i~upuść (sekcja~\ref{sec:panel-edit}).

\subsection{Pasek stanu}

Jednowierszowy pasek na dole okna wyświetla:
\begin{itemize}
    \item aktualny stan programu (np.~\textit{Gotowy}, \textit{Obliczanie\ldots}, \textit{Wczytywanie pliku\ldots}),
    \item liczbę wierzchołków i~krawędzi aktualnie wczytanego grafu,
    \item nazwę ostatnio użytego algorytmu lub trybu wczytywania.
\end{itemize}

\section{Formaty danych}

Aplikacja korzysta z~formatów danych zdefiniowanych przez silnik obliczeniowy (program C autorstwa Vyshni i~Udavichenki). Poniżej opisano każdy z~nich.

\subsection{Plik wejściowy -- lista krawędzi}
\label{sec:format-input}

Plik tekstowy z~listą krawędzi grafu. Każda linia reprezentuje jedną krawędź:

\begin{lstlisting}
<nazwa_krawedzi> <wierzcholek_A> <wierzcholek_B> <waga_krawedzi>
\end{lstlisting}

\textbf{Przykład:}
\begin{lstlisting}
AB   1  2  1
BC   2  3  1
CD   3  4  1
DB   4  2  1.407
\end{lstlisting}

Gdzie:
\begin{itemize}
    \item \texttt{nazwa\_krawedzi} -- ciąg znaków bez spacji (np.~\texttt{E1}, \texttt{AB}),
    \item \texttt{wierzcholek\_A}, \texttt{wierzcholek\_B} -- ciągi znaków identyfikujące węzły (np.~\texttt{1}, \texttt{A}),
    \item \texttt{waga\_krawedzi} -- liczba zmiennoprzecinkowa.
\end{itemize}

\subsection{Plik wyjściowy -- format tekstowy (.txt)}
\label{sec:fmt-txt}

Plik zawiera listę węzłów z~obliczonymi współrzędnymi. Brak nagłówka z~liczbą wierzchołków. Każda linia to jeden wierzchołek:

\begin{lstlisting}
<wierzcholek> <wspolrzedna_x> <wspolrzedna_y>
\end{lstlisting}

Współrzędne zapisywane są w~formacie \texttt{\%.2f} (dwa miejsca po przecinku).

\textbf{Przykład:}
\begin{lstlisting}
1 0.00 0.00
2 1.00 0.00
3 1.00 1.00
4 0.00 1.00
\end{lstlisting}

\subsection{Plik wyjściowy -- format binarny (.bin)}
\label{sec:fmt-bin}

Plik binarny bez nagłówka. Rekordy wierzchołków następują bezpośrednio po sobie do końca pliku. Każdy rekord ma zmienną długość:

\begin{itemize}
    \item \textbf{1 bajt} -- długość nazwy wierzchołka (\texttt{unsigned char}),
    \item \textbf{$n$ bajtów} -- nazwa wierzchołka (ciąg ASCII, bez znaku końca linii; $n$ to wartość poprzedniego bajtu),
    \item \textbf{8 bajtów} -- współrzędna $x$ (\texttt{double}, IEEE 754),
    \item \textbf{8 bajtów} -- współrzędna $y$ (\texttt{double}, IEEE 754).
\end{itemize}

Aplikacja Java odczytuje plik do końca strumienia, tworząc po jednym rekordzie wierzchołka na iterację.

\textbf{Przykład} (wierzchołek o~nazwie \texttt{"1"} i~współrzędnych $(0.0,\;0.0)$):
\begin{lstlisting}
[ 01 ][ 31 ][ 00 00 00 00 00 00 00 00 ][ 00 00 00 00 00 00 00 00 ]
\end{lstlisting}
Gdzie \texttt{01} to długość nazwy, \texttt{31} to kod ASCII znaku \texttt{'1'}, a~oba bloki 8-bajtowe to wartość \texttt{0.0} w~formacie IEEE 754.

\section{Dostępne algorytmy silnika obliczeniowego}
\label{sec:algorithms}

Silnik obliczeniowy udostępnia dwa algorytmy wybierane flagą \texttt{-a} przy wywołaniu:

\begin{center}
\renewcommand{\arraystretch}{1.5}
\begin{tabular}{>{\raggedright\arraybackslash}p{4cm} >{\raggedright\arraybackslash}p{9.5cm}}
\toprule
\textbf{Nazwa algorytmu} & \textbf{Opis} \\
\midrule
\texttt{fruchterman} & Algorytm siłowy (Fruchterman-Reingold). Symulacja fizyczna: węzły odpychają się jak cząstki naładowane, krawędzie przyciągają je jak sprężyny. Iteracyjne chłodzenie zapewnia równomierny i~czytelny układ. Algorytm domyślny. \\
\addlinespace
\texttt{triangulation} & Algorytm geometryczny. Zapobiega krzyżowaniu się krawędzi przez precyzyjne rozmieszczanie wierzchołków z~użyciem reguł triangulacji. Graf musi być planarny; w~przeciwnym razie silnik zgłasza błąd. \\
\bottomrule
\end{tabular}
\end{center}

% ---------------------------------------------------------------
\section{Przykładowe scenariusze użycia}

\subsection{Obliczenie układu i wyświetlenie grafu}

\begin{enumerate}
    \item Uruchom aplikację. Panel wizualizacji jest pusty (czarne tło).
    \item W~pasku menu kliknij \textbf{Ustawienia grafu} (aktywne domyślnie).
    \item W~polu \textit{Plik wejściowy} kliknij \textit{Przeglądaj} i~wskaż plik z~listą krawędzi (np.~\texttt{graf.txt}).
    \item Wybierz algorytm: \textit{Fruchterman-Reingold} lub \textit{Triangulacja}.
    \item Opcjonalnie wskaż katalog wyjściowy i~format zapisu.
    \item Kliknij \textbf{Wykonaj}. Aplikacja wywołuje silnik C z~odpowiednimi argumentami. Po zakończeniu obliczeń panel wizualizacji wyświetla graf.
\end{enumerate}

\subsection{Wczytanie gotowego pliku pozycji}

\begin{enumerate}
    \item W~zakładce \textit{Ustawienia grafu} wybierz opcję \textit{Plik tekstowy (.txt)} lub \textit{Plik binarny (.bin)} w~sekcji \textit{Wczytaj gotowy plik pozycji}.
    \item W~polu ścieżki kliknij \textit{Przeglądaj} i~wskaż plik pozycji.
    \item Kliknij \textbf{Wykonaj}. Aplikacja odczytuje plik i~wyświetla graf. Silnik C nie jest wywoływany.
\end{enumerate}

\subsection{Edycja pozycji wierzchołka przyciskami kierunkowymi}

\begin{enumerate}
    \item Po wyświetleniu grafu kliknij \textbf{Edycja} w~pasku menu.
    \item Z~listy rozwijanej wybierz identyfikator wierzchołka.
    \item Ustaw żądany krok przesunięcia w~polu \textit{Krok przesunięcia}.
    \item Klikaj przyciski $\uparrow\downarrow\leftarrow\rightarrow$, aby przesuwać wierzchołek. Panel wizualizacji odświeża się po każdym kliknięciu.
    \item Kliknij \textbf{Zatwierdź do pliku}, aby zapisać nowe współrzędne.
\end{enumerate}

\subsection{Edycja pozycji wierzchołka przez przeciągnij i upuść}

\begin{enumerate}
    \item Po wyświetleniu grafu kliknij i~przytrzymaj przycisk myszy na wybranym wierzchołku w~panelu wizualizacji.
    \item Przeciągnij wierzchołek w~nowe miejsce -- przesuwa się on na bieżąco wraz z~kursorem.
    \item Zwolnij przycisk myszy. Nowe współrzędne zostają automatycznie zapisane do pliku wyjściowego.
\end{enumerate}

\subsection{Zmiana ustawień wyświetlania}

\begin{enumerate}
    \item Kliknij \textbf{Ustawienia widoku} w~pasku menu.
    \item Włącz lub wyłącz \textit{Pokaż identyfikatory wierzchołków} oraz \textit{Pokaż wagi krawędzi}. Zmiany widoczne są natychmiast.
    \item Przesuń suwak \textit{Powiększenie}, aby zmienić skalę widoku.
    \item Kliknij \textit{Wyśrodkuj} lub \textit{Resetuj zoom} wedle potrzeby.
\end{enumerate}
